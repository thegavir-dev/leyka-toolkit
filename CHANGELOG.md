# Changelog

## 0.2.0

### Исправлено

- **Метка донора теперь присваивается корректно.** Корневая причина: Leyka создаёт WP-пользователя (роль «Донор») асинхронно — после хуков `leyka_new_donation_added` и даже после `transition_post_status`. Решение: назначение метки отложено на `shutdown`, когда Leyka уже завершила обработку. Кеш `get_user_by` сбрасывается через `wp_cache_delete()` перед поиском. Поддержаны оба сценария хранения доноров: WP users (основной) и `leyka_donor` post type (fallback).
- **Чекбокс подписки не отображался в шаблоне «star».** В star-шаблоне нет блока `.section__fields.agreements` — чекбоксы оферты/ПД находятся в `.donor__oferta span`. Добавлен дополнительный JS-селектор `.donor__oferta:not(.studioavp-leyka-addon-subscribe) span`. Исправлен fallback: `setProperty('display', 'block', 'important')` вместо `style.display = ''`, чтобы перебить CSS `!important`.

### Добавлено

- **Хук `on_donation_funded`.** Метка назначается при смене статуса донации на `funded` (callback от платёжного шлюза или ручная смена в админке). Приоритет 9999 — выполняется после всей логики Leyka.
- **Двойная стратегия поиска донора.** Сначала `get_user_by('email')` (WP users), затем `get_posts` по `leyka_donor` (если post type зарегистрирован).
- **Встроенный лог.** Пункт меню «Лог» с чекбоксом и кнопкой очистки. Файл `wp-content/uploads/leyka-toolkit/toolkit.log`, защищён `.htaccess`. Логирование по умолчанию выключено.

## 0.1.1

### Добавлено

- **Встроенный лог.** Новый пункт меню «Лог» с чекбоксом «Вести логирование» и кнопкой «Очистить лог». Лог хранится в `wp-content/uploads/leyka-toolkit/toolkit.log`, защищён `.htaccess`. Метод `handle_subscribe()` логирует каждый шаг: сохранение подписки, получение donor ID, проверку таксономии, создание и назначение метки. Логирование по умолчанию выключено.

## 0.1.0

### Исправлено

- **Метка донора не применялась.** Таксономия задана напрямую как `donors_tag` вместо обращения к константе `Leyka_Donor::DONORS_TAGS_TAXONOMY_NAME`, которая могла не совпадать. Убран несуществующий метод `get_donor()` — ID донора берётся из `$donation->donor_id` / `$donation->donor_user_id`. Термин создаётся автоматически через `wp_insert_term()`, если его ещё нет. Убран вызов `calculate_donor_metadata()`, который мог вызывать побочные ошибки. Добавлено логирование ошибок при создании и назначении метки.
- **Сохранение значения чекбокса подписки.** Метод `handle_subscribe()` сохраняет флаг подписки в штатный мета-ключ Leyka `leyka_donor_subscribed` и принудительно устанавливает `$donation->donor_subscribed = 1`, чтобы Leyka видела подписку сразу.
- **Проверка наличия Leyka при активации.** Если Leyka не установлена или не активна, плагин деактивируется и показывает сообщение об ошибке вместо того, чтобы работать вхолостую.
- **Проверка существования таксономии донора.** Перед `wp_set_object_terms()` добавлен вызов `taxonomy_exists()` — метка не записывается в несуществующую таксономию.
- **Зависимость фронт-скрипта от Leyka.** Inline-скрипт теперь зависит от `leyka-public` (если он зарегистрирован), что гарантирует корректный порядок выполнения при AJAX-рендере форм.

### Улучшено

- **Устойчивый DOM-селектор.** JS ищет целевой контейнер сначала по `[data-leyka-toolkit-target="subscribe"]`, затем по CSS-селектору `.section__fields.agreements .donor__oferta span` как fallback. Тема или плагин может пометить свой контейнер data-атрибутом — JS подхватит его вместо дефолтного селектора.
- **Фильтр для списка шаблонов.** Список поддерживаемых шаблонов вынесен в `apply_filters('leyka_toolkit_supported_templates', [...])`. Сторонний код может добавить свой шаблон без правки плагина.
- **Детерминированный field_id.** `wp_rand()` заменён на статический счётчик `$render_count` — стабильные ID при кешировании и в тестах.
- **Кеш настроек.** `settings_data()` читает `get_option()` один раз за запрос, результат хранится в `$settings_cache`. Кеш сбрасывается при сохранении настроек.
